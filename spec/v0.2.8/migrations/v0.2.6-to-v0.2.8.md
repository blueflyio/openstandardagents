# Migration Guide: v0.2.6 to v0.2.8

## Overview

v0.2.8 is a patch release focused on version synchronization fixes and developer experience improvements. No breaking schema changes.

**Note**: v0.2.7 was deprecated due to release issues. Skip directly from v0.2.6 to v0.2.8.

## Schema Changes

**No schema changes** - v0.2.8 uses the same schema structure as v0.2.6.

## What Changed

### Version Management
- Fixed `.version.json` synchronization
- Corrected package.json schema export paths
- Updated CHANGELOG with proper version entries

### Developer Experience
- Added comprehensive test suite
- Improved lint configuration
- Enhanced documentation

## Migration Steps

### Automatic Migration

No migration needed - v0.2.6 manifests work with v0.2.8:

```bash
# Your existing manifests work as-is
ossa validate my-agent.ossa.yaml
```

### Manual Updates (Optional)

Update `apiVersion` in your manifests:

```yaml
# Before (v0.2.6)
apiVersion: ossa/v0.2.6

# After (v0.2.8) - optional
apiVersion: ossa/v0.2.8
```

Both versions are compatible.

## Breaking Changes

**None** - This is a patch release.

## Deprecated Features

**None**

## New Features

- Enhanced test coverage
- Improved error messages
- Better version validation

## Upgrade Checklist

- [ ] Update `@bluefly/openstandardagents` to 0.2.8
- [ ] Run `ossa validate` on existing manifests
- [ ] Optionally update `apiVersion` to `ossa/v0.2.8`

## Rollback

If needed, rollback to v0.2.6:

```bash
npm install @bluefly/openstandardagents@0.2.6
```
