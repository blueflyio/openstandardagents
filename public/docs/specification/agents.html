Agent Taxonomy

OSSA defines 9 standard agent types, each with specific responsibilities and capabilities:

1. Workers

  • Execute specific, well-defined tasks
  • Stateless and horizontally scalable
  • Focus on single responsibility

2. Orchestrators

  • Coordinate multiple agents
  • Manage workflows and dependencies
  • Handle task distribution and aggregation

3. Integrators

  • Connect with external systems
  • Handle API communications
  • Transform data between formats

4. Monitors

  • Observe system health
  • Track performance metrics
  • Alert on anomalies

5. Critics

  • Review and validate output
  • Provide quality assessment
  • Suggest improvements

6. Judges

  • Make decisions between options
  • Resolve conflicts
  • Apply business rules

7. Governors

  • Enforce policies and compliance
  • Manage resource allocation
  • Control access and permissions

8. Voice

  • Handle natural language interfaces
  • Manage conversational flows
  • Provide user interaction layer

9. Trainers

  • Improve agent capabilities
  • Manage learning processes
  • Update knowledge bases

Implementation Standards

All OSSA-compliant agents must:

  • Implement the ACDL (Agent Communication & Discovery Language) protocol
  • Provide OpenAPI 3.1 specifications
  • Support health check endpoints
  • Emit standardized metrics
  • Handle graceful shutdown

Communication Protocols

Agents communicate using:

  • REST APIs for synchronous operations
  • WebSockets for real-time updates
  • Message Queues for asynchronous tasks
  • gRPC for high-performance inter-agent communication

Discovery Mechanism

The ACDL discovery protocol enables:

  • Automatic agent registration
  • Capability advertisement
  • Dynamic service mesh formation
  • Load balancing and failover
