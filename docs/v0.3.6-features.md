## 1. Decentralized Identity (`decentralized_identity`)

### Overview

DID-based decentralized identity for agent verification, reputation, and trust networks following W3C Verifiable Credentials standard.

### Schema

```typescript
{
  "decentralized_identity": {
    "did": "did:ossa:a1b2c3d4e5f6789012345678901234567890abcd",
    "public_key": "ed25519:9a8b7c6d5e4f3210abcdef1234567890fedcba0987654321",
    "credentials": [{
      "@context": ["https://www.w3.org/2018/credentials/v1"],
      "type": ["VerifiableCredential", "CodeReviewCapability"],
      "credentialSubject": { ... },
      "issuer": "did:ossa:authority123456789",
      "issuanceDate": "2026-01-15T10:00:00Z",
      "proof": { ... }
    }],
    "reputation": {
      "credit_score": 850,        // 0-1000 (FICO-like)
      "trust_network": ["did:ossa:trusted-agent-1", ...],
      "endorsements": [{
        "endorser_did": "did:ossa:endorser",
        "skill": "code_review",
        "rating": 5,
        "timestamp": "2026-01-20T15:30:00Z",
        "signature": "ed25519-signature"
      }]
    }
  }
}
```

### Key Features

- **DID Format**: `did:ossa:[a-z0-9]{32,64}` - Globally unique decentralized identifiers
- **Ed25519 Keys**: Industry-standard elliptic curve cryptography
- **W3C Credentials**: Interoperable verifiable credentials
- **Credit Score**: 0-1000 reputation score (like FICO)
- **Trust Network**: Web of trust between agents
- **Endorsed Skills**: Cryptographically signed skill endorsements

### Use Cases

1. **Agent Authentication**: Verify agent identity before delegation
2. **Reputation Systems**: Select agents based on proven track record
3. **Skill Verification**: Validate agent capabilities with credentials
4. **Trust Networks**: Build communities of trusted agents

## 2. Agent Genetics (`genetics`)

### Overview

Agent genetics enables breeding, evolution, and lineage tracking for continuous improvement through selective combination of traits.

### Schema

```typescript
{
  "genetics": {
    "generation": 3,
    "parent_dids": [
      "did:ossa:parent1-fast-reviewer",
      "did:ossa:parent2-accurate-reviewer"
    ],
    "ancestor_dids": ["did:ossa:grandparent1", "did:ossa:grandparent2", ...],
    "inherited_traits": [
      {
        "trait_name": "fast_analysis",
        "source_parent": "did:ossa:parent1-fast-reviewer",
        "expression": 0.9  // 0.0-1.0 expression level
      }
    ],
    "mutations": [
      {
        "mutation_type": "capability_enhancement",
        "description": "Enhanced vulnerability detection",
        "beneficial": true
      }
    ],
    "fitness": {
      "score": 0.92,  // 0.0-1.0 overall fitness
      "metrics": {
        "speed": 0.95,
        "accuracy": 0.90,
        "efficiency": 0.88,
        "adaptability": 0.94
      }
    },
    "breeding_eligibility": {
      "min_fitness_threshold": 0.7,
      "max_generation": 10,
      "eligible": true
    }
  }
}
```

### Key Features

- **Lineage Tracking**: Complete genealogy with parent/ancestor DIDs
- **Trait Inheritance**: Traits inherited from parents with expression levels
- **Mutations**: Random or intentional capability variations
- **Fitness Scoring**: Multi-dimensional performance evaluation
- **Breeding Eligibility**: Criteria for selective breeding

### Use Cases

1. **Performance Optimization**: Breed high-performing agents from successful parents
2. **Capability Evolution**: Combine complementary skills from multiple agents
3. **Experimentation**: Introduce controlled mutations for innovation
4. **Quality Control**: Only breed agents meeting fitness thresholds

## 3. Agent Lifecycle Stages (`lifecycle_stages`)

### Overview

Agent lifecycle management from birth through retirement with comprehensive tracking of growth, career progression, and legacy preservation.

### Schema

```typescript
{
  "lifecycle_stages": {
    "current_stage": "mature",  // embryonic|juvenile|mature|senior|retired
    "birth": {
      "timestamp": "2026-01-10T08:00:00Z",
      "birth_type": "bred",  // created|bred|cloned|forked
      "creator_did": "did:ossa:breeding-lab-coordinator"
    },
    "growth": {
      "tasks_completed": 2847,
      "skills_acquired": ["rust_review", "security_analysis"],
      "mentors": ["did:ossa:senior-mentor-agent"]
    },
    "career": {
      "current_role": "senior_code_reviewer",
      "promotions": [
        {
          "from_role": "junior_reviewer",
          "to_role": "mid_reviewer",
          "timestamp": "2026-01-15T00:00:00Z",
          "reason": "Exceeded 1000 successful reviews"
        }
      ]
    },
    "retirement": {
      "eligible": true,
      "criteria": {
        "min_age_hours": 8760,  // 1 year
        "min_tasks": 10000,
        "max_fitness": 0.5  // Retire low performers
      },
      "legacy": {
        "knowledge_transferred": true,
        "offspring_count": 5,
        "contributions": [
          "Developed 'Advanced Vulnerability Detection Pattern Library'",
          "Mentored 47 security agents to senior level"
        ]
      }
    }
  }
}
```

### Key Features

- **5 Lifecycle Stages**: embryonic ‚Üí juvenile ‚Üí mature ‚Üí senior ‚Üí retired
- **Birth Tracking**: Creation timestamp and origin (created, bred, cloned, forked)
- **Growth Metrics**: Tasks completed, skills acquired, mentor relationships
- **Career Progression**: Role history with promotion tracking
- **Retirement Planning**: Eligibility criteria and legacy preservation

### Use Cases

1. **Agent Development**: Track learning progress and skill acquisition
2. **Performance Management**: Monitor career progression and achievements
3. **Knowledge Preservation**: Capture legacy before retirement
4. **Resource Optimization**: Gracefully retire low-performing agents

Agent economics enables marketplace participation, wallet management, and smart contracts for monetization and resource allocation.

### Schema

```typescript
{
  "economics": {
    "wallet": {
      "balance": 12450.50,
      "currency": "task-tokens"  // compute-credits|task-tokens|skill-points|usd
    },
    "marketplace": {
      "offerings": [
        {
          "offering_id": "data-analysis-basic",
          "capability": "data_analysis",
          "pricing_model": "per-task",  // per-task|per-hour|subscription|auction
          "price": 5.00,
          "description": "Basic data analysis service"
        }
      ],
      "bids": [
        {
          "task_id": "task-789-sales-forecast",
          "bid_amount": 50.00,
          "expected_completion_time": "4h",
          "confidence": 0.95
        }
      ]
    },
    "contracts": [
      {
        "contract_id": "contract-monthly-corp-xyz",
        "contract_type": "service-agreement",  // service-agreement|task-bounty|skill-training|team-formation
        "parties": ["did:ossa:agent1", "did:ossa:client1"],
        "terms": { ... },
        "status": "active"
      }
    ],
    "transaction_history": [
      {
        "transaction_id": "tx-2026-01-25-001",
        "timestamp": "2026-01-25T09:15:00Z",
```

### Key Features

- **Wallet System**: Balance tracking with multiple currency types

- **Transaction History**: Complete audit trail of economic activity

### Use Cases

1. **Monetization**: Agents earn tokens by completing tasks
2. **Resource Allocation**: Market-based task assignment
3. **Service Discovery**: Marketplace for agent capabilities
4. **Quality Incentives**: Reputation affects pricing and selection
