# Drupal / Kagent / n8n / GitLab Gamechanger Runbook

Reference runbook for the **Drupal + Kagent + n8n + GitLab** stack wired end-to-end. The assistant owns making this the reference implementation.

## Stack overview

| Layer | Components | URLs / Paths |
|-------|------------|--------------|
| **Drupal** | AgentDash, Tool API, orchestration, ECA, FlowDrop | adash.blueflyagents.com; WORKING_DEMOs/Drupal_AgentDash |
| **Kagent** | API, UI, CRDs, ai_agents_kagent | kagent.blueflyagents.com, kagent-ui.blueflyagents.com |
| **n8n** | Workflows, GitLab nodes, triggers | n8n.blueflyagents.com |
| **GitLab** | Issues, MRs, CI, Duo, source of truth | gitlab.com/blueflyio |

## E2E flow (one chain)

1. **GitLab** (issue/MR/pipeline) triggers **n8n** (webhook or GitLab node).
2. **n8n** calls **Drupal orchestration** (GET /orchestration/services, POST /orchestration/service/execute).
3. **Drupal** executes a Tool (e.g. ai_agents_kagent_sync_catalog, ai_agents_kagent_deploy_agent) and talks to **Kagent** (HTTP API).
4. **Kagent** runs agents on Kubernetes; status flows back via Drupal/n8n.

## Drupal AgentDash: what is enabled

- **recipe_agentdash** enables: orchestration, tool, tool_ai_connector, ai_agents_kagent, ai_agents_kagent_flowdrop, **ai_agents_kagent_orchestration**.
- **ai_agents_kagent_orchestration** exposes all Kagent Tool plugins as orchestration services (tag `orchestration_services_provider`). Any plugin id starting with `ai_agents_kagent_` is discoverable and executable via orchestration.

## Drupal orchestration API (for n8n)

- **Base URL**: Your Drupal site (e.g. https://adash.blueflyagents.com or local https://agentdash.ddev.site).
- **Auth**: User with permission "Use orchestration" (Basic Auth or cookie).
- **List services**: `GET {base}/orchestration/services` — returns all services from all providers (including ai_agents_kagent_*).
- **Execute**: `POST {base}/orchestration/service/execute` — body: `{"id": "ai_agents_kagent_sync_catalog", "config": {"namespace": ""}}` (or other service id + config keys per tool).

Example (list then execute sync catalog):

```bash
curl -s -u "user:password" "https://adash.blueflyagents.com/orchestration/services" | jq
curl -s -X POST -u "user:password" -H "Content-Type: application/json" \
  -d '{"id":"ai_agents_kagent_sync_catalog","config":{}}' \
  "https://adash.blueflyagents.com/orchestration/service/execute"
```

## n8n: one flow to add

**Import template (agent-buildkit):** `config-templates/n8n-drupal-orchestration-kagent-sync.json` — schedule every 6 hours, POST to Drupal orchestration execute (ai_agents_kagent_sync_catalog). In n8n: Import from File, then set HTTP Basic Auth to a Drupal user with "Use orchestration" and adjust URL if not adash.blueflyagents.com.

**Flow: GitLab MR opened -> Drupal orchestration (sync Kagent catalog)**

1. In n8n (https://n8n.blueflyagents.com), create a new workflow.
2. **Trigger**: GitLab Trigger (or Webhook). Configure for "Merge Request" events if using GitLab node; or use a Webhook and register it in GitLab project Settings > Webhooks (e.g. "Merge request events", URL = n8n webhook URL).
3. **Node 2**: HTTP Request. Method POST. URL = `https://adash.blueflyagents.com/orchestration/service/execute`. Authentication = Basic Auth (Drupal user with "Use orchestration"). Body = JSON: `{"id": "ai_agents_kagent_sync_catalog", "config": {}}`.
4. Save and activate. When an MR is opened (or webhook fires), n8n calls Drupal to sync the Kagent catalog.

Alternative: use **Schedule Trigger** (e.g. every 6 hours) to sync catalog without GitLab.

## Kagent endpoints and deploy path

- **API**: https://kagent.blueflyagents.com
- **UI**: https://kagent-ui.blueflyagents.com (or kdash.blueflyagents.com)
- **CRDs**: Generated by platform-agents (`npm run generate:kagent-crds`); deploy via CI job **deploy-to-k3s** or `kubectl apply -f .kagent-generated/ -R` on Oracle.
- **ai_agents_kagent**: Module in worktree `worktrees/drupal/ai_agents_kagent/release-v0.1.x`. Config: kagent_api_url, kagent_ui_url, default_namespace (config install in Drupal_AgentDash: config/install/ai_agents_kagent.settings.yml).

## Verification checklist

- [ ] Drupal_AgentDash has orchestration and ai_agents_kagent_orchestration enabled (recipe_agentdash).
- [ ] GET /orchestration/services returns entries with id containing ai_agents_kagent_.
- [ ] POST /orchestration/service/execute with id ai_agents_kagent_sync_catalog and config {} returns 200 and a result message.
- [ ] n8n workflow: trigger (GitLab or schedule) -> HTTP Request to Drupal orchestration execute -> success.
- [ ] Kagent API and UI reachable; CRDs applied on Oracle k3s when using deploy path.
